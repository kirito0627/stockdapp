## 第一阶段：本地调试（Ganache）

> 目标：在你自己的电脑上跑通整个系统（后端 + 前端 + 合约），确认注册登录、买卖股票都能正常工作。

### 步骤 1：启动 Ganache

打开 Ganache，确保：
- 端口：**8545**
- Chain ID：**1337**
- 状态显示 Running

Ganache 会显示 10 个测试账户，每个账户有 100 ETH。**记住第一个账户的私钥**（点右边的钥匙图标可以看），后面要用。

### 步骤 2：部署合约到 Ganache

打开终端（命令行），执行：

```bash
cd C:\WorkBuddy\StockDApp
npx hardhat run scripts/deploy.js --network ganache
```

部署成功后，终端会输出类似：
```
MockUSDT deployed: 0x5513F...280A
StockMarket deployed: 0x708A8...338C
```

同时 `deployed-addresses.json` 会自动更新（已经改成自动同步到前端了）。

### 步骤 3：把合约地址填入后端 .env

打开 `C:\WorkBuddy\StockDApp\backend\.env`，把步骤 2 输出的地址填进去：

```env
RPC_URL=http://127.0.0.1:8545
DEPLOYER_PRIVATE_KEY=
USDT_ADDRESS=0x5513F...280A
MARKET_ADDRESS=0x708A8...338C
```

> 注意：`DEPLOYER_PRIVATE_KEY` 留空即可，后端会自动用 Ganache 第一个账户。

### 步骤 4：安装后端依赖并启动

```bash
cd C:\WorkBuddy\StockDApp\backend
npm install
npm start
```

看到以下输出说明成功：
```
Database initialized at .../stockdapp.db
Deployer wallet: 0x...
Contracts initialized.
  RPC_URL: http://127.0.0.1:8545
  USDT: 0x5513F...
  MARKET: 0x708A8...
StockDApp backend running on http://localhost:3001
```

> 这个终端窗口不要关，后端需要一直运行。

### 步骤 5：启动前端

新开一个终端窗口：

```bash
cd C:\WorkBuddy\StockDApp\frontend
npm install    # 第一次需要安装依赖
npm run dev
```

看到 `Local: http://localhost:5173/` 说明成功。

### 步骤 6：测试注册登录

1. 浏览器打开 `http://localhost:5173`
2. 看到登录/注册页面
3. 点"注册"标签，输入用户名（如 test）和密码（如 123456）
4. 点"注册并获取 100 万 USDT"
5. 注册成功后自动登录，进入主界面
6. 试试买入股票、查看持仓

**如果注册失败：**
- 检查 Ganache 是否在运行
- 检查后端终端有没有报错
- 检查 `.env` 里的合约地址是否正确
- 检查 `deployed-addresses.json` 里的地址和 `.env` 是否一致

## 第二阶段：部署到 Sepolia 测试网

> 目标：把合约部署到以太坊 Sepolia 测试网，这样不需要本地 Ganache，任何人都能通过 RPC 访问。

### 步骤 1：准备一个钱包

你需要一个以太坊钱包（MetaMask 或任何 EVM 钱包），里面有一些 Sepolia 测试 ETH。

**领取免费 Sepolia ETH：**
1. 打开 https://cloud.google.com/application/web3/faucet/ethereum/sepolia
2. 输入你的钱包地址
3. 免费领取 0.5 ETH（足够部署合约）

或者搜索 "sepolia faucet" 找其他免费水龙头。

**导出私钥：**
- MetaMask → 账户详情 → 导出私钥（需要输入密码）
- 把私钥复制下来（格式：`0xabc123...`）

### 步骤 2：配置 hardhat.config.js

`hardhat.config.js` 已经配好了 sepolia 网络，它会读取环境变量：

```js
sepolia: {
  url: process.env.RPC_URL || "https://ethereum-sepolia.publicnode.com",
  accounts: process.env.PRIVATE_KEY ? [process.env.PRIVATE_KEY] : []
}
```

### 步骤 3：部署合约到 Sepolia

在终端设置环境变量并部署（Windows PowerShell）：

```powershell
cd C:\WorkBuddy\StockDApp
$env:PRIVATE_KEY="0x你的私钥"
npx hardhat run scripts/deploy.js --network sepolia
```

> 部署到 Sepolia 需要等待几分钟（每个交易都要等区块确认）。
> 198 只股票分 4 批添加，总共可能需要 5-10 分钟。

部署成功后，终端会输出新的合约地址（跟 Ganache 的不同）。

### 步骤 4：更新后端 .env

```env
RPC_URL=https://ethereum-sepolia.publicnode.com
DEPLOYER_PRIVATE_KEY=0x你的私钥
USDT_ADDRESS=0x新的USDT地址
MARKET_ADDRESS=0x新的StockMarket地址
```

### 步骤 5：重启后端测试

```bash
cd C:\WorkBuddy\StockDApp\backend
npm start
```

注册一个新用户测试。这次注册会在 Sepolia 链上创建钱包和铸币，可能需要等 10-20 秒。

### 步骤 6：同步前端合约地址

```bash
copy C:\WorkBuddy\StockDApp\deployed-addresses.json C:\WorkBuddy\StockDApp\frontend\public\deployed-addresses.json
```

## 第三阶段：部署到云平台（公网访问）

> 目标：别人通过网址就能访问你的 DApp，不需要你电脑开机。

### 步骤 1：部署后端到 Railway（免费）

1. 打开 https://railway.app 注册账号（可以用 GitHub 登录）
2. 点 "New Project" → "Deploy from GitHub repo"
3. 连接你的 GitHub，把 StockDApp 项目推到 GitHub
4. 在 Railway 设置：
   - Root Directory: `backend`
   - Build Command: `npm install`
   - Start Command: `npm start`
5. 添加环境变量（在 Railway 的 Variables 页面）：
   ```
   RPC_URL=https://ethereum-sepolia.publicnode.com
   DEPLOYER_PRIVATE_KEY=0x你的私钥
   USDT_ADDRESS=0x你的USDT地址
   MARKET_ADDRESS=0x你的StockMarket地址
   JWT_SECRET=stockdapp-jwt-secret-2024
   ENCRYPTION_KEY=stockdapp-secret-key-2024
   PORT=3001
   ```
6. 部署成功后，Railway 会给你一个网址，如：
   `https://stockdapp-backend.up.railway.app`

测试：浏览器打开 `https://stockdapp-backend.up.railway.app/api/health`
应该返回 `{"status":"ok",...}`

### 步骤 2：部署前端到 Vercel（免费）

1. 在 `frontend` 目录创建 `.env` 文件：
   ```
   VITE_API_URL=https://stockdapp-backend.up.railway.app/api
   ```

2. 打开 https://vercel.com 注册账号（可以用 GitHub 登录）

3. 点 "New Project" → 导入 GitHub 仓库

4. 设置：
   - Root Directory: `frontend`
   - Framework Preset: Vite
   - Build Command: `npm run build`
   - Output Directory: `dist`
   - Environment Variables:
     ```
     VITE_API_URL=https://stockdapp-backend.up.railway.app/api
     ```

5. 部署成功后，Vercel 会给你一个网址，如：
   `https://stock-dapp.vercel.app`

### 步骤 3：分享网址

把这个网址发给任何人，他们打开就能：
1. 注册/登录（自动获得 100 万 USDT）
2. 看行情、K线图、涨跌榜
3. 买卖股票
4. 查看持仓和交易记录

**不需要安装任何东西，不需要 MetaMask，不需要懂区块链。**

## 常见问题

### Q: 注册时报 "网络错误，请检查后端服务是否启动"
A: 后端没启动。执行 `cd backend && npm start`

### Q: 注册时报 "mint failed" 或 "insufficient funds"
A: Ganache 模式下，合约地址填错了。检查 `.env` 里的地址和 `deployed-addresses.json` 是否一致。

### Q: Sepolia 部署时报 "insufficient funds for gas"
A: 你的钱包没有足够的 Sepolia ETH。去 faucet 免费领取。

### Q: 前端打开了但显示空白
A: 检查浏览器控制台（F12）的报错信息。通常是后端没启动或 API 地址不对。

### Q: 每次重启 Ganache 都要重新部署吗？
A: 是的。Ganache 重启后所有数据清空，需要重新部署合约并更新 `.env` 里的地址。Sepolia 模式不需要。
